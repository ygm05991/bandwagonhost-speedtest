## 三种搬瓦工测速方法，从简单到专业

### 方法一：直接 ping 测试 IP（最快）

这是最省事的方法。把上表的测试 IP 直接拿来 ping，两三秒就能看出延迟。

**Windows 用户：**

ping 89.208.246.192

打开 cmd 或 PowerShell，把 IP 换成你要测的机房 IP 即可。

**Mac / Linux 用户：**

ping 89.208.246.192

打开终端，一样的命令。

想测全国各地到某个机房的延迟？去 ping.chinaz.com 输入 IP，会自动跑全国几十个城市的 ping 结果，一眼看出自己地区的延迟水平。

---

### 方法二：SpeedTest 网页测速（最直观）

Ping 只能测延迟，看不到实际下载速度。SpeedTest 演示站把下载速度、上传速度、延迟都测出来。

打开对应机房的演示站链接（见上表），点击测试，等个30秒。

DC6 和 DC9 这两个 CN2 GIA 机房都有演示站，是测试最常见的两个选择。从过往实测结果来看，DC6 CN2 GIA-E 机房三网回程全走 CN2 GIA，电信、联通、移动均为优质线路，平均延迟在200ms以内，电信更低。DC9 在晚高峰期间电信和联通的下载速度也能保持在较高水平，丢包率接近零。

### 方法三：LookingGlass 路由跟踪（最专业）

这个方法适合认真研究线路质量的用户。LookingGlass 可以做 MTR、traceroute、ping 测试，能看出你的数据包从机房出来之后走了哪些节点、有没有绕路。

**如何使用：**

1. 进入演示站，找到 LookingGlass 入口（通常是 `/lg/` 路径，比如 `https://dc6.bwg.net/lg/`）
2. 输入你本地的 IP 地址（页面通常会有"您的IP"按钮，点一下自动填入）
3. 选择测试类型，比如 traceroute 或 MTR
4. 点击"开始测试"，查看每一跳的路由节点信息

看到 `59.43.XX.XX` 这种 IP 出现，就说明走了 CN2 GIA 线路。这个 IP 段属于中国电信的国际出口 CN2 节点，出现它就代表线路质量有保证。

## 用 Speedtest-CLI 在 VPS 内部测速

已经买了搬瓦工、想测试 VPS 本身的带宽和速度？不用 SpeedTest 网页，直接在 Linux 命令行安装 Speedtest-CLI 更准确。

**安装步骤如下：**

1. SSH 登录搬瓦工 VPS
2. 安装 Speedtest-CLI（以 Debian/Ubuntu 为例）：

bash
curl -s https://packagecloud.io/install/repositories/ookla/speedtest-cli/script.deb.sh | sudo bash
sudo apt-get install speedtest


3. 运行测速命令：

bash
speedtest


4. 查看输出的下载速度、上传速度和延迟数据

工具会自动选择离 VPS 最近的测速节点。如果想测到国内节点的速度，可以手动指定节点 ID（`speedtest --list` 可以查看可用节点列表）。
